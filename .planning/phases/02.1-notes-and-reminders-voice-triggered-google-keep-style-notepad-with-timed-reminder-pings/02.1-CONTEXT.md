# Phase 02.1: Notes and Reminders — Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Voice-triggered notepad and reminder system. Users say "Hey Jarvis, add note: [content]" to create notes, or "remind me to [action] [time]" to create time-bound reminders. Notes and reminders are stored in SQLite and displayed in a Google Keep-style Midnight Glass dashboard accessible from the right-rail icon strip. Reminders fire with in-app sound + browser Web Notification and stay overdue until dismissed.

Voice scope for this phase: **add note** and **add reminder** only. Delete, list, and complex reminder management are deferred to v2.

</domain>

<decisions>
## Implementation Decisions

### Dashboard placement & entry point
- New icon in the right-rail (RSB) icon strip opens Notes as a full-screen overlay
- Same pattern as Finance panel — replaces main content area, closes via icon click or Escape
- A dedicated `register_notes()` module following the existing router registration pattern

### Card layout
- Masonry/waterfall grid layout (not a flat list) — closest to Google Keep
- Cards support: color tinting and pin-to-top — no labels/tags in v1
- Clicking an empty dashed card in the grid enters inline creation mode (Google Keep style)
- Cards are click-to-edit inline — click anywhere on an existing card to enter edit mode, save on click-away
- Trash icon appears on card hover; click deletes with a 5-second "Undo" toast

### Card colors — Midnight Glass dark tints
- 6 dark translucent tints tuned to Midnight Glass (all use `backdrop-filter: blur + saturate`):
  - Default (near-black glass)
  - Deep blue
  - Dark teal
  - Amber night
  - Rose night
  - Mauve glass
- Colors are subtle dark tints — not Google Keep's bright rainbow

### Reminders — visual distinction
- Reminder cards show a clock badge (⏰) + formatted due date/time at bottom of card
- Amber card tint + amber clock badge when overdue (past due_at, not yet dismissed)
- Reminder stays in the grid with overdue state until user explicitly dismisses or deletes it

### Voice command parsing
- Router LLM parses note/reminder intent and returns structured JSON:
  `{type: "note"|"reminder", content: "...", due_at: "ISO8601 or null"}`
- LLM receives current datetime in router system prompt context for relative time resolution
- "tomorrow" with no time specified → defaults to 9:00 AM next day
- Full natural language resolution: "in 2 hours", "next Friday at noon", "end of day" all handled by LLM
- Timezone: server local timezone (host machine system timezone)

### Voice confirmation after note/reminder saved
- Localis speaks a short TTS confirmation: "Got it. Note saved." or "Reminder set for [time]."
- A chat message also appears in main chat: brief confirmation with note preview

### Voice intents (v1 scope)
- "add note: [content]" → plain note
- "remind me to [action] [time]" → note with due_at
- No voice delete in v1 (voice-only: add note and add reminder)

### Reminder delivery mechanism
- Frontend JS polls `GET /notes/due` every 30 seconds for fired-but-undismissed reminders
- On trigger: plays in-app chime sound + fires browser Web Notification API (OS-level popup)
- Web Notification requires one-time permission request on first reminder (handled gracefully if denied)
- Badge on notes icon in RSB strip shows count of pending/overdue reminders
- Overdue card in notes grid glows/pulses amber

### Notes in chat context
- New tool: `notes.retrieve` — router can invoke it when user asks about notes in chat
- Notes are injected as a clean tool result into system prompt context (same pattern as `memory.retrieve`)
- NOT added to conversation history — avoids hallucination from irrelevant prior messages
- All notes are available for retrieval (full content, not opt-in tagged subset)
- Example: "Hey Jarvis, what did I note about YT Music?" → router calls notes.retrieve → LLM answers from injected content

### Backend schema — notes table
```sql
CREATE TABLE IF NOT EXISTS notes (
    id TEXT PRIMARY KEY,
    content TEXT NOT NULL,
    note_type TEXT DEFAULT 'note',   -- 'note' | 'reminder'
    due_at TEXT,                     -- ISO8601 UTC, null for plain notes
    color TEXT DEFAULT 'default',    -- color key name
    pinned INTEGER DEFAULT 0,        -- 0 | 1
    dismissed INTEGER DEFAULT 0,     -- for reminders: 0 = active, 1 = dismissed
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
)
```

### Claude's Discretion
- Exact chime audio file (short, subtle system-tone feel — not jarring)
- Exact CSS for masonry grid layout (CSS columns or JS masonry library)
- Exact card hover animation and trash icon position
- SQL query for notes.retrieve (full-text or most recent N notes)
- Web Notification icon and body format
- Exact amber glow/pulse CSS animation for overdue cards

</decisions>

<specifics>
## Specific Ideas

- User's exact phrasing: "Hey Jarvis, add note: lets add a yt music integration into localis using HA" — content after "add note:" is the full note body
- User's exact phrasing: "remind me to order pasta from instacart tomorrow" — no explicit time, defaults to 9 AM next day
- Notes in chat context: "the model should only get system prompt + tier 1 memory + user prompt + notes via RAG — it should not hallucinate because of previous context or other irrelevant info so it stays focused"
- Dashboard visual: Google Keep masonry grid, but dark Midnight Glass palette — not Google Keep's bright colors

</specifics>
