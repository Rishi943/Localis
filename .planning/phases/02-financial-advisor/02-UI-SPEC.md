## Layout Contract

The Finance panel is a **full-viewport overlay** (`position: fixed; inset: 0; z-index: 200`) that contains an internal 3-column layout matching the Notion "Simple Finance Tracker" reference. It is NOT a separate page.

```
┌─────────────────────────────────────────────────────────┐
│  Finance Header: tabs | period selector | refresh | upload | close │
├──────────────────┬──────────────────────────────────────┤
│  Left sidebar    │  Center-top: line chart | donut chart │
│  (Budgets)       ├──────────────────────────────────────┤
│  8 category rows │  Center-bottom: Expenses list         │
│                  │  (grouped by month, collapsible)      │
└──────────────────┴──────────────────────────────────────┘
```

- Left sidebar width: **240px fixed**, no collapse, scrollable if content overflows
- Center column: `flex: 1`, contains chart row (top half) and expenses list (bottom half)
- Chart row: always **side-by-side** — line chart left, donut chart right; no responsive stacking
- Chat tab: replaces entire center area with finance chat pane (left sidebar hidden in Chat tab)
- Panel open/close animation: `opacity` + `transform: translateY(8px)` → `translateY(0)`, duration 200ms, `ease-out`

**Source:** CONTEXT.md "3-column layout" decision, Notion "Simple Finance Tracker" reference.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, chart label gaps |
| sm | 8px | Compact element spacing, tab inner padding, row gaps, budget form grid gap |
| md | 16px | Default panel padding, section content spacing, transaction row padding horizontal |
| lg | 24px | Panel section breaks, chart vertical rhythm |
| xl | 32px | Header-to-content spacing within Finance panel |
| 2xl | 48px | Major section breaks in Dashboard |
| 3xl | 64px | Not used in Finance panel |

Exceptions:
- Bar chart track height: 8px (budget sidebar progress bars)
- Budget progress bar: height 6px (slightly thinner than category bars)
- Touch targets on tab buttons: minimum 44px height
- Transaction row min-height: 40px (`padding: 8px 16px` on each side)
- Month group header: `padding: 12px 16px` (12px is not on grid — use 16px/8px split: `padding: 8px 16px` to stay on-grid; visual grouping via background tint, not extra padding)

**Source:** Spacing derived from existing `app.css` patterns (modal 16px 18px, RSB section padding). 8-point grid applied for all Finance-specific elements.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 15px | 400 | 1.5 | Finance Chat tab messages (matches `.msg-text` at 15px/1.5 in app.css) |
| Label | 12px | 600 | 1.4 | Section headers, category names, tab labels, column headers, badge text, period label, "No budget" text, month group headers in expenses list, source tag text |
| Data | 14px | 400 | 1.3 | Transaction descriptions, period selector, upload CTA text, amount column, budget row spend amounts, onboarding chat input |
| Display | 24px | 600 | 1.0 | Total spend figures, key dashboard numbers (not currently used in V2 — reserved for future summary card if added) |

Weights declared: **400** (body, data) and **600** (labels, display). Two weights only.

Mono font (JetBrains Mono, size 12px, weight 400) used for: date column in transaction list and bar amount labels in budget sidebar. Same label scale (12px) but mono face only.

**Consolidation note:** 4 sizes maximum: 12px (label), 14px (data), 15px (body), 24px (display). No intermediate sizes (10px, 11px, 13px) for new `.fin-*` elements. Existing pre-V2 `.fin-*` rules in app.css that use 11px or 13px sizes are V1 artifacts — V2 rewrites those selectors to 12px or 14px.

**Source:** UIUX/DESIGN.md (Inter 400/600), app.css `.msg-text` 15px/1.5, `.rsb-scard-val` display pattern.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgba(15, 15, 15, 0.45)` via `--bg-panel` | Finance panel background, all `.fin-*` glass surfaces |
| Secondary (30%) | `rgba(255, 255, 255, 0.05)` via `--card-bg` | Transaction list row tint, chart area background, onboarding chat bubbles, tab bar background, budget sidebar background |
| Accent (10%) | `#5A8CFF` via `--accent-primary` | See reserved list below |
| Credit indicator | `#4ade80` (confirmed in existing app.css `.fin-tx-credit`) | Credit transaction amount text and "↑ Credit" tag in expenses list only |
| Destructive | `#ef4444` via `--status-red` | "Reset goals" button hover/confirm state; over-budget bar fill (actual > budget) |
| Amber (budget warning) | `#f59e0b` | Budget progress bar fill at 85–100% of budget |

**Accent (`--accent-primary: #5A8CFF`) reserved for:**
1. Active tab underline / active tab text (2px bottom border + text color change)
2. Budget sidebar progress bar fill — green state only (actual < 85% of budget)
3. "Upload CSV" primary CTA button background
4. Period selector dropdown focus ring
5. Chart.js line chart stroke color
6. Chart.js donut chart primary segment (largest spending category)
7. Chart.js donut chart remaining segments: 7 opacity/hue-shifted tints in the same blue family — `rgba(90, 140, 255, 0.85)` through `rgba(90, 140, 255, 0.15)` stepped evenly — not Chart.js default rainbow palette
8. Onboarding "Skip for now" link text on hover

Accent is NOT used for: transaction row hover, tab backgrounds, chart track fills, body text, "Bank" source tag, "Credit Card" source tag, or general interactive states.

**Source tags (expenses list):**
- "Bank" pill: `background: rgba(74, 222, 128, 0.12); color: #4ade80; border: 1px solid rgba(74, 222, 128, 0.2)` — green family
- "Credit Card" pill: `background: rgba(90, 140, 255, 0.12); color: #5A8CFF; border: 1px solid rgba(90, 140, 255, 0.2)` — blue/accent family

**Budget progress bar states (budget sidebar):**
- < 85% of budget: fill = `var(--accent-primary)` (#5A8CFF)
- 85–100% of budget: fill = `#f59e0b` (amber)
- > 100% of budget: fill = `var(--status-red)` (#ef4444), bar capped at 100% width, amount text also turns red

**No-budget state (budget sidebar):** Row shows actual spend with a dashed/ghost progress bar (`border: 1px dashed rgba(255,255,255,0.12); background: transparent`). "No budget" label in `var(--text-muted)` inline right of spend amount. Still shows actual spend amount.

**Source:** UIUX/DESIGN.md and app.css `:root` variables. Credit indicator `#4ade80` from existing `.fin-tx-credit` rule confirmed in app.css line 1105. Destructive `#ef4444` from `--status-red` CSS variable at line 62. Amber `#f59e0b` is Claude's discretion per CONTEXT.md.

## Component Inventory

All new Finance V2 components use the `.fin-` prefix. V1 `.fin-*` rules in app.css are rewritten in-place. No existing non-finance class names are modified.

### Finance Panel Overlay (`.fin-panel`)

- `position: fixed; inset: 0; z-index: 200` — above RSB (z-index: 100, confirmed in STATE.md), below settings overlay (z-index: 300, confirmed in actual app.css inspection noted in STATE.md)
- Glass recipe: `background: var(--bg-panel); backdrop-filter: var(--glass-filter); -webkit-backdrop-filter: var(--glass-filter); border: 1px solid var(--border-subtle); box-shadow: 0 16px 40px rgba(0,0,0,0.5), inset 0 1px 0 var(--border-highlight)`
- Toggled via `.hidden` class (not DOM removal)
- Open/close animation: `opacity 0s` + `transform: translateY(8px)` → `translateY(0)`, duration 200ms, `ease-out`
- ESC key closes panel (listener registered on open, removed on close)

### Finance Header Bar (`.fin-header`)

- Full-width bar at top of panel: `display: flex; align-items: center; padding: 8px 16px; border-bottom: 1px solid var(--border-subtle)`
- Left section: Tab bar (`.fin-tabs`)
- Right section: period selector + refresh button + "Upload CSV" trigger + close button
- All right-section controls in `.fin-header-actions` flex row with 8px gap

### Tab Bar (`.fin-tabs`)

- Two tabs: "Dashboard" (default) and "Chat"
- Active tab: `border-bottom: 2px solid var(--accent-primary)`, text color `var(--text-primary)`, weight 600
- Inactive tab: text color `var(--text-secondary)`, weight 400, no border
- Tab height: 44px minimum (accessibility)
- Tab background: transparent; `.fin-header` provides the visual container

### 3-Column Dashboard Layout (`.fin-dashboard-body`)

- `display: flex; flex-direction: row; flex: 1; overflow: hidden`
- Left column: `.fin-budget-sidebar` — `width: 240px; flex-shrink: 0; border-right: 1px solid var(--border-subtle); overflow-y: auto; padding: 16px`
- Center column: `.fin-center` — `flex: 1; display: flex; flex-direction: column; overflow: hidden`
  - `.fin-charts` top half: `display: flex; flex-direction: row; gap: 16px; padding: 16px; border-bottom: 1px solid var(--border-subtle)` — line chart left, donut right, always side-by-side
  - `.fin-tx-pane` bottom half: `flex: 1; overflow-y: auto; padding: 0 16px 16px`

### Budget Sidebar (`.fin-budget-sidebar`)

- Section header: "Budgets" at 12px/600, `var(--text-secondary)`, uppercase with letter-spacing, `margin-bottom: 16px`
- "Reset goals" ghost button: inline below header, 12px `var(--text-muted)`, no background, hover → `var(--status-red)`
- 8 category rows always shown (Food, Transport, Shopping, Utilities, Entertainment, Health & Fitness, Government & Fees, Other)
- Each row: category name (12px/600 `var(--text-secondary)`), spend amount (12px mono `var(--text-primary)`), budget amount (12px mono `var(--text-muted)`), progress bar
- Progress bar track: `height: 6px; background: rgba(255,255,255,0.06); border-radius: 4px; overflow: hidden`
- Progress bar fill: `height: 100%; width: calc(var(--pct, 0) * 1%); border-radius: 4px; transition: width 0.4s cubic-bezier(0.4, 0, 0.2, 1)` — color varies by state (see Color section)
- No-budget row: dashed ghost progress bar + "No budget" label (see Color section)
- "Reset goals" confirmation: inline text below button — "This clears your budgets and restarts setup. Reset?" with "Yes, reset" (red) and "Cancel" inline links — no modal

- X-axis: distinct months with transaction data (`strftime('%Y-%m', date)` formatted as "Mar 2026") — no gap-filling for months with no data
- Y-axis: total debit spend per month (CAD, no decimal in tick labels)
- Line color: `var(--accent-primary)` (#5A8CFF), fill: `rgba(90, 140, 255, 0.08)` area under curve
- Gridlines: `rgba(255, 255, 255, 0.06)` on y-axis, none on x-axis
- Tooltip: dark glass style — `backgroundColor: 'rgba(15,15,15,0.9)', borderColor: 'rgba(255,255,255,0.12)', titleColor: '#fff', bodyColor: 'rgba(255,255,255,0.7)'`
- Chart title: "Monthly Spend" at 12px/600 `var(--text-secondary)` uppercase, positioned above canvas (not via Chart.js title plugin)
- Empty state: canvas remains in layout, centered ghost text "No data yet — upload a statement to get started" at 12px `var(--text-muted)` — implemented as an overlay div, not Chart.js options

## Interaction States

### Panel Open/Close

- Open: `financeUI.open()` removes `.hidden`, triggers `_checkOnboarding()`
- Close: `financeUI.close()` adds `.hidden`
- ESC key closes Finance panel (add keydown listener on open, remove on close)

### Tab Switching

- Click tab → remove `.active` from current pane/tab button, add to target
- No transition on tab content pane — instant swap (consistent with existing RSB tab pattern)
- Chat tab: hides `.fin-budget-sidebar`, shows `.fin-chat-pane` full-width

### Chart Loading

- On period selector change or refresh: set `--pct` to 0 on all budget bars, re-fetch from API, then:
  - Budget sidebar bars: set `--pct` to actual values — CSS `transition: width 0.4s` handles animation
  - Chart.js charts: call `chart.data.datasets[0].data = [...]` + `chart.update('active')` — Chart.js handles animation natively
- Loading state: budget bars remain at 0; chart area shows `opacity: 0.4` on existing render

### Upload Flow

- File selected + account label filled → "Upload" button enables
- Submit → "Uploading..." text, button disabled
- Success → button restores, success message shown for 4 seconds, expenses list and charts refresh
- Error → error message shown, button restores
- Period selector repopulates if new months were added

### Onboarding Navigation

- Step advances by index — no NLP parsing; the active step index determines which answer key is stored
- "Skip for now" → sets local skip flag, shows Dashboard (empty state if no uploads)
- Onboarding completion → `fin_onboarding_done` set via `/finance/goals`, shows Dashboard

### Budget Sidebar Hover

- Category row hover: `background: rgba(255,255,255,0.03)` on the row
